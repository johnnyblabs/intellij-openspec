## Why

OpenSpec CLI 1.6.0's `openspec update` changes the tracked agent-skill surface: the tracked `openspec-*` skill set grows from five to six (new `openspec-update-change/`, backing the `/opsx:update` agent workflow — agent-skill only, no CLI subcommand), and every generated `SKILL.md` is stamped with `allowed-tools` and `generatedBy` frontmatter. Auditing the plugin's own understanding of that surface found a real gap: the Explore feature's skill-instruction lookup still reads only pre-1.5 command paths (`.claude/commands/opsx/explore.md` and mirror equivalents), which no longer exist on any skills-era (1.5+) project — so project-customized explore instructions are silently ignored and the built-in fallback always wins.

Tracker: this change is linked to an existing tracker issue via the gitignored `.tracking.yaml` sidecar (per the repository's tracker-sidecar convention).

## What Changes

- `ExplorePromptService` reads the skills-era path first (`.claude/skills/openspec-explore/SKILL.md`), keeping the legacy command paths as fallbacks for pre-1.5 projects; the existing frontmatter stripping applies to the 1.6 `allowed-tools`/`generatedBy` stamps.
- This repository's own tracked `openspec-*` skills are regenerated by the real 1.6.0 `openspec update` (5 → 6 directories, stamped frontmatter) — the only CLI-regenerated tracked surface.
- Value-level 1.6 sanity checks recorded (no code change needed): the multi-line requirement `text` in show/change-show JSON has no plugin consumer, and task-progress aggregation only changes for nested-tasks schemas (the plugin reads a change's single `tasks.md`, which is unchanged).
- No doc-enumeration updates required: an audit found no plugin doc enumerating the five-skill set.

## Capabilities

### New Capabilities

None.

### Modified Capabilities

- `explore-context`: the explore skill-instruction lookup requirement changes — the skills-era location SHALL be searched first, with the legacy command paths retained as fallbacks.

## Impact

- **Code**: `ExplorePromptService.SKILL_FILE_PATHS` (one path prepended).
- **Tests**: `ExplorePromptServiceTest` gains a skills-era-path case (skills path wins over legacy when both exist; frontmatter stripped).
- **Repo files**: `.claude/skills/openspec-*` regenerated at 1.6.0 (six dirs).
- **Docs**: user-facing CHANGELOG entry (Explore honors project skill customizations on skills-era projects). No platform-API impact.
